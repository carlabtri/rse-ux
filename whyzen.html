Whyzen

A graphical user interface for Energy & Material's Accelerated Materials Design and Discovery (AMDD) team to create, save, load, and analyze graphical models — using a node-based graph editor for building and visualizing mechanism models.

▶ Demo✦ Figma⬡ Jira Epic

Overview

A graphical user interface for the E&M AMDD team to create, save, load, and analyze graph-theory based models — using a node-based graph editor that interfaces with TRI-AMDD's existing ML codebase for training.

Goals

Serves as a prototype for a causal modeling tool that enables non-AI/ML subject matter experts to integrate their domain expertise with fully interpretable and explainable graphical models. These models offer more robust and safer causal reasoning than LLMs or correlative feature analyses, and can extrapolate to unseen domains more effectively than purely data-driven methods.

Approach

  • Multiple iterative design sprints
  • Focused on discoverability and interaction clarity
  • Close collaboration with engineering on feasibility

Model Creation & Editing

A node-based graph editor for quickly creating, visualizing, modifying, and studying graph-theory based mechanism models. Designed to be powerful and precise while remaining discoverable for non-UX engineers.

Save & Load

Saving and loading interfaces directly with TRI-AMDD's existing ML codebase, enabling seamless integration between the visual editor and the training pipeline.

Analysis

Tools to analyze graphical models and explore their causal structures — supporting the team's goal of building interpretable, explainable models that reason causally rather than correlationally.

Causal Reasoning

Unlike LLMs or correlative feature analyses, the causal models built with Whyzen offer more robust and safer reasoning capabilities — and can extrapolate to unseen domains more effectively than purely data-driven methods.
